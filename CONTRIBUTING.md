# Contributing

Thanks for considering a contribution to BagInCoffee.

## Workflow

1. Fork the repository.
2. Create a branch for your change.
3. Make focused commits with clear messages.
4. Run the relevant app, web, or backend checks before opening a PR.
5. Open a pull request with a short summary and screenshots when UI changes are involved.

## Project Areas

- `BagInCoffee-App/`: Flutter mobile client
- `BagInCoffee-Web/`: SvelteKit web client
- `BagInDB/`: Rust/Axum backend

## Notes

- Keep environment secrets out of commits.
- Update README or setup docs when behavior or configuration changes.
- Prefer small pull requests over broad refactors.
